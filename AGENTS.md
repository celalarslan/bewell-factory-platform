# Agent Rules

1. Every agent must read PROJECT_ROADMAP.md before changing code.
2. GitHub main is the official source of truth.
3. Do not invent features, endpoints, database models, or business rules.
4. Do not make large refactors without explicit approval.
5. Do not modify files outside the assigned task.
6. Never print or commit secrets.
7. Do not delete production or user data.
8. Do not run database reset, seed, truncate, or destructive migrations without explicit approval.
9. Every code change must include proportionate tests.
10. Do not commit or push when build or required tests fail.
11. At the end of every task report:
   - changed files
   - commands and tests
   - remaining risks
   - git status
12. PROJECT_ROADMAP.md may only be changed for verified architecture or approved product decisions.
